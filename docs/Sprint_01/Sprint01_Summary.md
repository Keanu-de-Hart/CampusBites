## Sprint 1 Overview

**Sprint Duration:** 30 March 2026 - 13 April 2026

---

### ✅ Completed User Stories

* User authentication (login & registration with role-based access)
* Role-based dashboards (customer and vendor)
* Homepage (index page) with navigation and hero section
* Customer menu browsing functionality
* Vendor menu management (add, update status, delete items)

---

### 📋 Summary of Work Completed

During this sprint, the team successfully implemented the core foundation of the system. This includes user authentication using Firebase, role-based access control, and the development of separate dashboards for customers and vendors.

The homepage was designed and developed with a responsive layout, including navigation and a hero section to improve user experience.

Additionally, key platform features were introduced:

* Vendors can create and manage menu items, including updating availability (e.g., sold out) and deleting items
* Customers can browse menu items from multiple vendors and view item details

---

### 🧪 Testing

* Basic unit testing implemented using Jest
* Continuous Integration (CI) configured using GitHub Actions
* Core logic (e.g., validation and role handling) tested

---

### 👨‍💻 Contributions

* Frontend development (homepage, dashboards, UI components)
* Backend integration with Firebase (authentication and Firestore)
* Implementation of role-based navigation and access control
* Setup of testing environment and CI pipeline

### ⚠️ Challenges
